JavaScript 常见错误与解决方案

JavaScript 开发中常见的错误类型及调试方法，帮助快速定位和解决问题。

常见错误类型一览

错误类型 错误信息 原因
ReferenceError xxx is not defined 使用了未定义的变量
TypeError Cannot read property 'xxx' of undefined 访问 undefined/null 的属性
SyntaxError Unexpected token 语法错误
RangeError Invalid array length 数组长度无效
URIError URI malformed URI 编码/解码错误
EvalError EvalError eval() 使用错误

1. ReferenceError - 未定义变量

// 错误：使用了未声明的变量 console.log(myVar); // ReferenceError: myVar is not defined // 正确：先声明再使用 let myVar = 'hello'; console.log(myVar); // 'hello' // 错误：变量作用域问题 function test() { console.log(x); // ReferenceError let x = 1; } test(); // 正确：先声明 function test() { let x = 1; console.log(x); // 1 }

2. TypeError - 类型错误

// 错误：访问 null/undefined 的属性 const obj = null; obj.name; // TypeError: Cannot read property 'name' of null const arr; arr[0]; // TypeError: Cannot read property '0' of undefined // 正确：使用可选链 const obj = null; console.log(obj?.name); // undefined，不会报错 // 正确：使用 && 短路运算 const name = obj && obj.name; // 错误：调用非函数 const fn = 'hello'; fn(); // TypeError: fn is not a function // 正确：检查类型 if (typeof fn === 'function') { fn(); }

3. 异步错误处理

// Promise 未捕获错误 fetch('/api/data') .then(res => res.json()) .then(data => console.log(data)) .catch(err => console.error(err)); // 捕获错误 // async/await 错误处理 async function fetchData() { try { const res = await fetch('/api/data'); const data = await res.json(); return data; } catch (err) { console.error('请求失败:', err); throw err; } } // 全局未捕获 Promise 错误 window.addEventListener('unhandledrejection', function(event) { console.error('未捕获的 Promise 错误:', event.reason); });

4. 常见逻辑错误

错误类型 错误示例 正确写法
== 与 === if (a == null) if (a === null || a === undefined)
数组比较 [] === [] [].length === [].length
对象比较 {} === {} JSON.stringify(a) === JSON.stringify(b)
浮点数运算 0.1 + 0.2 === 0.3 Math.abs(0.1+0.2-0.3) < 0.0001
循环中的闭包 for(var i=0;i<3;i++){setTimeout(()=>console.log(i))} for(let i=0;i<3;i++){setTimeout(()=>console.log(i))}
